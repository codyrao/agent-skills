# Page.md 文档生成规则

## 概述
本文档定义了 Page.md 的生成规则，Page.md 是项目的页面接口文档，描述每个页面，包括页面路由、功能、页面参数信息等。主要面向前端和客户端，对于 API SDK 和后端可以不生成。

## 文档结构

### 1. 文档标题
- **要求**：清晰、简洁的文档标题
- **格式**：一级标题 `# 页面文档`

- **示例**：
```markdown
# 页面文档
```

### 2. 页面分组
- **要求**：按功能模块分组页面
- **格式**：二级标题 `## 模块名`

- **示例**：
```markdown
## 用户模块

## 订单模块

## 商品模块
```

### 3. 页面文档

#### 基本格式
```markdown
### 页面名称

**路由**：/path/to/page

**功能**：页面功能描述

**页面参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| param1 | string | 是 | 参数说明 |
| param2 | int | 否 | 参数说明 |

**页面交互**：

说明页面的交互逻辑和用户操作流程。

**注意事项**：

说明页面的注意事项和限制。
```

#### 详细格式
```markdown
### 页面名称

**路由**：/path/to/page

**功能**：页面功能描述

**页面参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| param1 | string | 是 | 参数说明 |
| param2 | int | 否 | 参数说明 |

**页面组件**：

| 组件名 | 说明 |
|--------|------|
| 组件1 | 组件说明 |
| 组件2 | 组件说明 |

**页面交互**：

说明页面的交互逻辑和用户操作流程。

**API 调用**：

| API | 说明 |
|-----|------|
| GET /api/users | 获取用户列表 |
| POST /api/users | 创建用户 |

**注意事项**：

说明页面的注意事项和限制。

**权限要求**：

说明页面的权限要求。
```

### 4. 分析项目页面

#### 前端项目（React/Vue/Angular）
分析以下内容：
- 路由配置文件（如 `router.js`、`routes.ts` 等）
- 页面组件文件（如 `pages/`、`views/` 目录下的文件）
- 页面路由定义

**示例分析（React Router）**：
```javascript
const routes = [
  {
    path: '/users',
    component: UserList,
  },
  {
    path: '/users/:id',
    component: UserDetail,
  },
  {
    path: '/users/create',
    component: UserCreate,
  },
];
```

**生成的页面文档**：
```markdown
## 用户模块

### 用户列表页面

**路由**：/users

**功能**：展示用户列表，支持分页、搜索、筛选

**页面参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | int | 否 | 页码，默认为 1 |
| size | int | 否 | 每页数量，默认为 10 |
| keyword | string | 否 | 搜索关键词 |

**页面组件**：

| 组件名 | 说明 |
|--------|------|
| UserTable | 用户表格组件 |
| SearchBar | 搜索栏组件 |
| Pagination | 分页组件 |

**页面交互**：

1. 用户进入页面，自动加载第一页数据
2. 用户可以输入关键词进行搜索
3. 用户可以点击分页按钮切换页面
4. 用户可以点击用户名进入用户详情页面
5. 用户可以点击"创建用户"按钮进入创建用户页面

**API 调用**：

| API | 说明 |
|-----|------|
| GET /api/users | 获取用户列表 |

**注意事项**：

- 搜索关键词支持模糊匹配
- 分页大小最大为 100

**权限要求**：

- 需要登录
- 需要用户列表查看权限

### 用户详情页面

**路由**：/users/:id

**功能**：展示用户详细信息，支持编辑、删除

**页面参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | int | 是 | 用户 ID |

**页面组件**：

| 组件名 | 说明 |
|--------|------|
| UserInfo | 用户信息组件 |
| UserActions | 用户操作组件 |

**页面交互**：

1. 用户进入页面，自动加载用户详细信息
2. 用户可以点击"编辑"按钮进入编辑用户页面
3. 用户可以点击"删除"按钮删除用户

**API 调用**：

| API | 说明 |
|-----|------|
| GET /api/users/:id | 获取用户详情 |
| DELETE /api/users/:id | 删除用户 |

**注意事项**：

- 删除用户需要二次确认
- 删除用户后返回用户列表页面

**权限要求**：

- 需要登录
- 需要用户详情查看权限
- 删除操作需要用户管理权限

### 创建用户页面

**路由**：/users/create

**功能**：创建新用户

**页面参数**：

无

**页面组件**：

| 组件名 | 说明 |
|--------|------|
| UserForm | 用户表单组件 |

**页面交互**：

1. 用户填写用户信息
2. 用户点击"提交"按钮创建用户
3. 创建成功后跳转到用户列表页面

**API 调用**：

| API | 说明 |
|-----|------|
| POST /api/users | 创建用户 |

**注意事项**：

- 用户名必须唯一
- 邮箱必须唯一
- 密码长度至少 6 位

**权限要求**：

- 需要登录
- 需要用户创建权限
```

## 文档放置位置
- 放在项目当前路径下 `./Page.md`

## 适用范围
- 主要面向前端和客户端
- 对于 API SDK 和后端可以不生成

## 注意事项
- 描述每个页面，包括页面路由、功能、页面参数信息
- 分析项目代码确认页面路由和功能
- 保持页面文档的清晰和详细
- 使用表格格式展示页面参数和 API 调用，提高可读性
- 说明页面的交互逻辑和用户操作流程
- 说明页面的注意事项和限制
- 说明页面的权限要求
- 及时更新页面文档，反映最新的页面变更
